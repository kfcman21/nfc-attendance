# 🌐 kfcman.link/attend 배포 가이드 (Oracle Cloud + Nginx)

크롬북에서 **경고 없이 PWA 설치**가 되도록, 출석 서버를 `https://kfcman.link/attend/` 서브경로에 올립니다.
정식 인증서(kfcman.link)가 그대로 적용되고, **Nginx 기본 인증(비밀번호)**으로 사이트를 보호합니다.

```
[크롬북: HID 리더기 + Chrome PWA]
        │  https://kfcman.link/attend/   (정식 인증서 + 기본 인증)
        ▼
[Oracle Cloud: Nginx]  ──/attend/ 접두어 제거──▶  [Node 서버 127.0.0.1:3100]
                                                         └─ SQLite (/var/lib/nfc-attendance)
```

> ⚠️ **전제**
> - **Node 22.5+** 필요 (앱이 `node:sqlite` 내장 모듈 사용). `node -v`로 확인.
> - 서버엔 리더기가 없음 → 카드 인식은 **크롬북에 꽂은 HID 리더기**가 담당(브라우저 입력 → 서버로 전송).
> - 메인 웹앱이 쓰는 내부 3000 포트는 건드리지 않음. 출석 서버는 **3100** 사용.

## 1. 코드 배치 & 의존성 설치 (Oracle 서버)
```bash
sudo mkdir -p /var/www/nfc-attendance
# 저장소(nfc 폴더)를 /var/www/nfc-attendance 로 복사 또는 git clone
cd /var/www/nfc-attendance
npm ci --omit=dev          # express, serialport, selfsigned 설치
sudo mkdir -p /var/lib/nfc-attendance
sudo chown -R www-data:www-data /var/www/nfc-attendance /var/lib/nfc-attendance
```
> serialport 네이티브 모듈이 빌드 안 되면(드묾) `NFC_DISABLE_SERIAL=1`로 어차피 안 쓰지만, import 자체가 실패하면
> `npm rebuild serialport` 또는 빌드 도구(`build-essential`, `python3`) 설치 후 재시도하세요.

## 2. 서비스 등록 (systemd)
```bash
sudo cp deploy/nfc-attend.service /etc/systemd/system/nfc-attend.service
# 필요 시 ExecStart 의 node 경로를 `which node` 결과로 수정
sudo systemctl daemon-reload
sudo systemctl enable --now nfc-attend
sudo systemctl status nfc-attend          # active(running) 확인
curl -s http://127.0.0.1:3100/api/status  # 내부 응답 확인
```

## 3. 기본 인증 비밀번호 생성
```bash
sudo apt-get install -y apache2-utils      # htpasswd 도구 (이미 있으면 생략)
sudo htpasswd -c /etc/nginx/.htpasswd-attend teacher   # 'teacher' 계정 생성, 비밀번호 입력
# 계정 추가는 -c 없이:  sudo htpasswd /etc/nginx/.htpasswd-attend another
sudo chown www-data:www-data /etc/nginx/.htpasswd-attend
```

## 4. Nginx 설정 추가
`deploy/nginx-attend.conf` 의 두 location 블록을 **기존 `kfcman.link` server 블록 안**에 붙여넣습니다.
(별도 server 블록이 아니라, 기존 `/on`, `/` 등과 같은 레벨)
```bash
sudo nginx -t        # 문법 검사
sudo systemctl reload nginx
```

## 5. 확인 (크롬북)
1. 크롬북 USB에 **HID 리더기** 연결
2. Chrome에서 `https://kfcman.link/attend/` 접속 → 기본 인증 아이디/비밀번호 입력
3. 주소창 오른쪽 **설치(⊕)** → "설치" (정식 인증서라 경고·플래그 불필요)
4. 화면에서 **HID 리더기 입력** 켜고 카드 태그 → 출석 처리

## 업데이트(재배포)
```bash
cd /var/www/nfc-attendance && git pull   # 또는 새 코드 복사
npm ci --omit=dev
sudo systemctl restart nfc-attend
```
DB는 `/var/lib/nfc-attendance`에 있어 재배포해도 유지됩니다.

## 참고 — 신뢰 인증서가 주는 차이
- 이 방식(정식 인증서)에서는 크롬북 설치(⊕) 버튼이 **경고 없이** 바로 표시됩니다.
- 반대로 LAN 단독(자체 서명) 방식은 경고를 한 번 허용해야 하고 설치 버튼이 안 뜰 수 있습니다.
  → 자세한 LAN 방식은 `docs/크롬북_PWA_설치_가이드.md` 참고.
