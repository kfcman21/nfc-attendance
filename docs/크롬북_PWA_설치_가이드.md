# 🟢 크롬북(ChromeOS)에서 학생 출석 앱 사용하기 (PWA)

크롬북은 Windows용 설치 파일(.exe)을 실행할 수 없습니다. 대신 **출석 서버는 Windows PC(또는 서버)에서 실행**하고,
**크롬북에서는 Chrome으로 접속해 "앱(PWA)"으로 설치**해서 사용합니다. 카드 인식은 **HID 키보드형 리더기**를 크롬북 USB에 꽂아 사용합니다.

```
[HID 리더기] → USB → [크롬북: Chrome PWA] ──네트워크──> [Windows PC: 출석 서버 + DB]
```

---

## 1단계. 서버 실행 (Windows PC)

1. 기존 `NFC학생출석-x.x.x-설치.exe`(또는 포터블)를 평소처럼 실행합니다.
2. 프로그램이 켜지면 내부 서버가 함께 뜹니다. 콘솔/로그에 접속 주소가 표시됩니다:
   ```
   이 PC에서:          http://localhost:3000
   같은 네트워크(크롬북): http://192.168.0.10:3000   ← 이 주소를 크롬북에서 사용
   ```
3. **방화벽 허용**: 처음 실행 시 Windows 방화벽이 물어보면 "허용"을 누릅니다.
   안 떴다면 인바운드 규칙에서 TCP **3000** 포트를 허용하세요.
4. PC와 크롬북이 **같은 Wi-Fi/네트워크**에 있어야 합니다.

> 💡 카드 인식 리더기는 **크롬북에 꽂습니다**(학생이 태그하는 자리). 서버 PC에는 리더기가 없어도 됩니다.

---

## 2단계. 크롬북에서 접속

1. 크롬북 Chrome 주소창에 위에서 본 주소를 입력: `http://192.168.0.10:3000`
2. 출석 화면이 뜨면 연결 성공입니다.

---

## 3단계. HID 리더기 켜기

1. HID 키보드형 NFC 리더기를 크롬북 USB에 연결합니다. (카드번호를 키 입력처럼 보내는 방식)
2. 출석 화면에서 **HID 리더기 입력**을 켭니다. (화면 하단/설정의 HID 토글)
3. 카드를 대면 번호가 자동 입력되어 출석 처리됩니다. 별도 드라이버가 필요 없습니다.

---

## 4단계. "앱"으로 설치하기 (PWA)

크롬북에 앱처럼 설치하면 전체화면 창·런처 아이콘으로 실행됩니다.

> ⚠️ **중요 — 보안 컨텍스트:** 브라우저 PWA 설치/오프라인 기능은 **HTTPS** 또는 **localhost**에서만 동작합니다.
> 그래서 서버는 **HTTPS(기본 포트 3443)** 를 함께 제공합니다. 크롬북에서는 `https://...:3443` 주소로 접속하세요.

### 방법 A — 서버 내장 HTTPS로 접속 (권장)
서버는 시작 시 HTTPS를 자동으로 켭니다. 콘솔/로그에 함께 표시됩니다:
```
HTTPS(크롬북 설치용): https://localhost:3443
  같은 네트워크:      https://192.168.0.10:3443   ← 크롬북에서 이 주소
  ※ 자체 서명 인증서 — 크롬북 첫 접속 시 보안 경고를 한 번 허용하세요.
```
1. **방화벽**: TCP **3443** 포트도 인바운드 허용(3000과 함께).
2. 크롬북 Chrome에서 `https://192.168.0.10:3443` 접속.
3. 자체 서명 인증서이므로 첫 접속에 **"연결이 비공개로 설정되어 있지 않습니다"** 경고가 뜹니다 →
   **고급 → (안전하지 않음)으로 이동**을 눌러 한 번 허용합니다. 이후 보안 컨텍스트로 인식됩니다.
4. 설치: Chrome 주소창 오른쪽 **설치(⊕) 아이콘** → "설치". (아래 참고)

> 🔐 **설치 버튼이 안 보이면 (신뢰된 인증서 사용):** 자체 서명 인증서는 "주의 요함"으로 표시돼 설치 버튼이 안 뜰 수 있습니다(서비스워커·오프라인은 동작). 설치 버튼까지 확실히 띄우려면 **신뢰된 인증서**를 쓰세요:
> - PC에서 `config.json`(또는 데이터 폴더의 `settings.json`)의 `https.cert`, `https.key`에 인증서·키 파일 경로를 지정하면 그 인증서로 HTTPS를 제공합니다.
> - 사설 CA 도구 **mkcert**로 발급한 인증서(개인/비관리 크롬북에 루트 CA 설치 가능할 때) 또는 학교 도메인의 **정식 인증서**, **Cloudflare Tunnel/ngrok** 같은 터널의 https 주소를 사용하면 설치 버튼이 정상 표시됩니다.

### 방법 B — (대안) LAN HTTP 주소를 "안전한 출처"로 등록 (크롬북마다 1회)
HTTPS 대신 평문 HTTP(3000)로 설치하고 싶을 때:
1. 크롬북 Chrome에서 `chrome://flags/#unsafely-treat-insecure-origin-as-secure` 열기
2. 입력칸에 서버 주소 입력: `http://192.168.0.10:3000`
3. **Enabled**로 바꾸고 **Relaunch**(재시작)
4. 다시 접속하면 보안 컨텍스트로 인식되어 서비스워커 등록 + 설치가 가능해집니다.

### 방법 C — 설치 없이 "앱 창"으로 띄우기 (가장 간단)
설치/오프라인 캐시는 없지만 앱처럼 별도 창으로 쓸 수 있습니다.
1. Chrome 우상단 ⋮ → **도구 더보기** → **바로가기 만들기**
2. **"창으로 열기"** 체크 → 만들기 → 런처에 아이콘 생성

## 자주 묻는 질문

- **인터넷이 끊겨도 되나요?** 출석 데이터는 서버 PC에 저장되므로, 크롬북↔서버 PC 간 네트워크는 연결돼 있어야 합니다.
  (PWA 오프라인 캐시는 화면 셸만 보관하고, 실제 출석 기록은 서버로 전송됩니다.)
- **여러 크롬북에서 동시에 쓸 수 있나요?** 네. 모두 같은 서버 주소로 접속하면 한 곳(서버 DB)에 기록됩니다.
- **서버 PC를 끄면?** 크롬북에서 접속이 안 됩니다. 출석을 받는 동안에는 서버 PC가 켜져 있어야 합니다.
- **리더기를 시리얼(CR-100) 방식으로 직접 쓰고 싶어요.** 현재 크롬북 버전은 HID 키보드형 리더기를 기준으로 합니다.
  Web Serial로 CR-100을 브라우저에서 직접 읽는 방식은 별도 개발이 필요합니다.
